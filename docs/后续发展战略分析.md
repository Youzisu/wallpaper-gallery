## 🎬 视频系列深度分析

### ✅ 优势

1. **内容丰富性高**
   - 动态壁纸市场需求存在（Windows 10+、macOS、Android）
   - 短视频素材库需求增长（自媒体、视频制作者）
   - 内容形式多样（风景延时、抽象动画、游戏 CG）

2. **差异化竞争**
   - 目前专注视频的公益平台较少
   - 大部分平台体验差、广告多、需要付费
   - 技术门槛高，竞争对手少

3. **技术挑战价值**
   - 展示流媒体技术能力
   - 适合作为技术作品集
   - 可学习 HLS、DASH、WebRTC 等前沿技术

### ❌ 劣势与挑战

#### 1. GitHub 存储限制

```
限制说明：
- 单文件最大：100MB
- 单仓库建议：< 1GB
- 实际问题：
  - 1080p 视频（10 秒）≈ 10-20MB
  - 4K 视频（10 秒）≈ 40-80MB
  - 100 个视频 ≈ 1-2GB（接近/超过限制）
```

**解决方案**：
- 使用外部存储（腾讯云 COS、阿里云 OSS）
- 月成本：¥20-50（100GB 存储 + 100GB 流量）

#### 2. jsDelivr CDN 限制

```
限制说明：
- 单文件限制：50MB
- 月流量配额：免费版有限制
- 实际问题：
  - 视频播放消耗流量极快
  - 假设日访问 1000 次，平均观看 3 个视频
  - 每视频 15MB → 45MB/用户
  - 日流量：1000 × 45MB = 45GB
  - 月流量：≈ 1.35TB
```

**解决方案**：
- 使用专业视频 CDN（七牛云、腾讯云）
- 月成本：¥100-300（取决于流量）

#### 3. 技术架构需彻底重构

**当前架构**：
```
GitHub Repository
    ↓
jsDelivr CDN
    ↓
静态网站（直接加载图片）
```

**视频所需架构**：
```
对象存储（COS/OSS）
    ↓
视频转码服务（生成多清晰度）
    ↓
视频 CDN（支持流媒体协议）
    ↓
HLS/DASH 播放器
```

**需要新增的技术栈**：
- 视频转码：FFmpeg、阿里云 MTS
- 流媒体协议：HLS（.m3u8）、DASH
- 播放器：Video.js、DPlayer、西瓜播放器
- 自适应码率：根据网速自动切换清晰度

#### 4. 用户体验问题

1. **移动端流量顾虑**
   - 用户担心消耗移动流量
   - 需要明确标注视频大小
   - 需要 WiFi 提醒功能

2. **加载等待时间**
   - 视频预加载需要时间
   - 用户等待耐心有限
   - 需要精心设计加载动画

3. **浏览器兼容性**
   - Safari、Chrome、Firefox 对视频格式支持不同
   - 需要提供多种格式（MP4、WebM）
   - 增加开发和存储成本

### 💰 成本估算

#### 存储成本
```
假设：100 个视频，平均 20MB
- 总存储：2GB
- 腾讯云 COS：¥0.099/GB/月 × 2GB ≈ ¥0.2/月
- 备份存储：+¥0.2/月
- 小计：¥0.4/月（几乎可忽略）
```

#### CDN 流量成本
```
假设：日访问 1000 人，人均观看 3 个视频
- 日流量：1000 × 3 × 20MB = 60GB
- 月流量：60GB × 30 = 1800GB ≈ 1.8TB
- 腾讯云 CDN：¥0.18/GB（前 50TB）
- 月成本：1800GB × ¥0.18 = ¥324/月
```

#### 转码成本
```
假设：每月上传 20 个新视频
- 转码规格：生成 3 种清晰度（480p、720p、1080p）
- 腾讯云 MTS：¥0.03/分钟
- 单视频 10 秒，3 个规格 = 30 秒 = 0.5 分钟
- 月成本：20 × 0.5 × ¥0.03 = ¥0.3/月（可忽略）
```

#### 总成本
```
存储 + CDN + 转码 = ¥0.4 + ¥324 + ¥0.3 ≈ ¥325/月

如果日访问量降低到 100 人：
CDN 成本 ≈ ¥32/月
总成本 ≈ ¥33/月
```

### 🎯 视频系列结论

**适合场景**：
- ✅ 有稳定收入来源支撑运营成本
- ✅ 日访问量已达到 5000+，有用户基础
- ✅ 愿意学习和维护复杂的流媒体架构
- ✅ 有明确的视频内容来源（合作或用户上传）

**不适合场景**：
- ❌ 项目刚起步，没有稳定流量
- ❌ 预算有限，无法承担月均 ¥50+ 运营成本
- ❌ 希望快速验证产品，获得用户反馈

#### 2. 表情包处理脚本

```bash
#!/bin/bash
# scripts/emoji-process.sh

# 表情包专用配置
EMOJI_SIZE=512              # 标准尺寸 512x512
EMOJI_QUALITY=90            # 质量 90%
EMOJI_FORMAT="png"          # 格式（PNG 支持透明）
EMOJI_WATERMARK_SIZE=36     # 水印大小（更小、更透明）
EMOJI_WATERMARK_OPACITY=20  # 水印透明度 20%

# 处理单个表情包
process_emoji() {
    local input="$1"
    local output="$2"

    # 1. 调整大小到 512x512（保持宽高比，填充空白）
    convert "$input" \
        -resize "${EMOJI_SIZE}x${EMOJI_SIZE}" \
        -background transparent \
        -gravity center \
        -extent "${EMOJI_SIZE}x${EMOJI_SIZE}" \
        "$output"

    # 2. 添加小水印（右下角）
    composite -gravity southeast \
        -geometry +10+10 \
        -dissolve ${EMOJI_WATERMARK_OPACITY} \
        watermark/emoji-watermark.png \
        "$output" "$output"

    echo "✅ 处理完成: $output"
}

# 批量处理表情包系列
process_series() {
    local series="$1"  # 例如：panda、xiaolian
    local input_dir="emoji_source/${series}"
    local output_dir="emoji/${series}"

    mkdir -p "$output_dir"

    for img in "$input_dir"/*.{png,jpg,jpeg,gif}; do
        [ -e "$img" ] || continue
        filename=$(basename "$img" | sed 's/\.[^.]*$/.png/')
        process_emoji "$img" "$output_dir/$filename"
    done
}

# 主函数
main() {
    if [ $# -eq 0 ]; then
        echo "用法: $0 <series>"
        echo "示例: $0 panda"
        exit 1
    fi

    process_series "$1"
}

main "$@"
```

#### 3. 元数据管理系统

```javascript
// scripts/generate-emoji-metadata.js

import fs from 'fs'
import path from 'path'
import crypto from 'crypto'

/**
 * 为表情包生成元数据
 */
async function generateEmojiMetadata(seriesPath) {
  const emojis = []
  const files = fs.readdirSync(seriesPath)

  for (const file of files) {
    const filePath = path.join(seriesPath, file)
    const stats = fs.statSync(filePath)

    // 生成唯一 ID（基于文件内容）
    const content = fs.readFileSync(filePath)
    const hash = crypto.createHash('md5').update(content).digest('hex')
    const id = hash.substring(0, 12)

    // 从文件名提取信息
    const name = path.parse(file).name

    // 自动标签（后续可接入 AI 识别）
    const tags = inferTags(name)

    emojis.push({
      id,
      name,
      filename: file,
      tags,
      size: `${(stats.size / 1024).toFixed(1)}KB`,
      format: path.extname(file).substring(1).toUpperCase(),
      created_at: new Date().toISOString(),
      usage_count: 0
    })
  }

  return emojis
}

/**
 * 从名称推断标签
 */
function inferTags(name) {
  const tagMap = {
    '无语': ['搞笑', '无奈'],
    '摸鱼': ['职场', '搞笑'],
    '加班': ['职场', 'emo'],
    '社死': ['搞笑', '尴尬'],
    // ... 更多映射
  }

  for (const [keyword, tags] of Object.entries(tagMap)) {
    if (name.includes(keyword)) {
      return tags
    }
  }

  return ['其他']
}
```

```javascript
export function selectCDN(fileSize, currentUsage) {
```
